# API Reference

ZYX exposes two embedding interfaces — a modern C++ API and a C-compatible API for FFI bindings.

## Which API to use?

- **C++ API** (`include/zyx/zyx.hpp`) — Use when developing in C++20. Offers type-safe values, RAII transaction management, and a fluent `QueryBuilder`.
- **Node.js API** (`npm install zyxdb`) — Use when developing in Node.js. Async interface with prebuilt platform binaries.
- **Python API** (`pip install zyxdb`) — Use when developing in Python. Provides Pythonic interface with context managers, iterators, and pandas integration.
- **C API** (`include/zyx/zyx_c_api.h`) — Use when building Rust, Go, or WASM bindings. Opaque handles with explicit lifecycle management.

## Design Conventions

- Query execution returns a `Result` / `ZYXResult_T*`. Callers iterate rows explicitly.
- Transactions are available in three forms:
  - **API-level explicit transactions** — `beginTransaction()` / `zyx_begin_transaction()`
  - **Read-only transactions** — `beginReadOnlyTransaction()` / `zyx_begin_read_only_transaction()`
  - **Cypher transaction statements** — `BEGIN` / `COMMIT` / `ROLLBACK` within `execute()`
- When no explicit transaction is active, `execute()` auto-commits write queries and uses a read-only fast path for reads.
