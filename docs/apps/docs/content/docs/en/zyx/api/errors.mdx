# Error Handling

ZYX uses two complementary error reporting mechanisms depending on the operation.

## 1. Exceptions (Lifecycle Operations)

Methods that cannot return a `Result` throw `std::exception` (or its subclasses) on failure:

| Method | Failure scenario |
|--------|-----------------|
| `Database::open()` | File I/O error, corrupt header |
| `Database::openIfExists()` | File does not exist, corrupt header |
| `Transaction::commit()` | WAL sync failure, storage flush failure |
| `Transaction::rollback()` | Storage I/O error during undo |

```cpp
try {
    zyx::Database db("/path/to/db");
    auto tx = db.beginTransaction();
    tx.execute("CREATE (n:Person {name: 'Alice'})");
    tx.commit();
} catch (const std::exception& e) {
    std::cerr << "Error: " << e.what() << std::endl;
}
```

Catch `std::exception` at your application boundary. ZYX exceptions carry descriptive `what()` messages.

## 2. Failed Result (Query Execution)

`Database::execute()` and `Transaction::execute()` **do not throw** for query-level errors. Instead, the error is captured in the `Result`:

```cpp
auto result = db.execute("INVALID CYPHER");
if (!result.isSuccess()) {
    std::cerr << "Query failed: " << result.getError() << std::endl;
}
```

| Result method | Description |
|---------------|-------------|
| `isSuccess()` | Returns `false` if the query failed |
| `getError()` | Returns the error message string |

Common query failure reasons:

- **Parse error** — Invalid Cypher syntax
- **Execution error** — Runtime failure during operator execution
- **Parameter error** — Missing or type-mismatched parameters

## Recommended Pattern

```cpp
zyx::Database db("/path/to/db");

auto tx = db.beginTransaction();
try {
    auto result = tx.execute(
        "CREATE (n:Person {name: $name})", {{"name", std::string("Alice")}});

    if (!result.isSuccess()) {
        // Query-level error — no exception thrown
        std::cerr << "Query error: " << result.getError() << std::endl;
        // tx will auto-rollback on destruction
        return;
    }

    tx.commit();
} catch (const std::exception& e) {
    // Lifecycle error (WAL sync, storage failure, etc.)
    std::cerr << "System error: " << e.what() << std::endl;
}
```
