# 状态链与并发控制

## 状态链（StateChain）

`StateChainManager` 用于存储可能超过单条 State 容量的数据：

- 可拆分为多条 State 链式存储
- 也可切换为 Blob 外部存储模式

主要操作：

- `createStateChain`
- `readStateChain`
- `updateStateChain`
- `deleteStateChain`

## 并发控制（当前实现）

并发核心在 `TransactionManager`：

- 写事务：独占锁（单写）
- 只读事务：共享锁 + committed snapshot

这与“传统 MVCC 多写并发”不同，当前策略更偏工程稳态：用单写降低冲突复杂度，用快照保障读一致性。
