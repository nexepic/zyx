# WAL 恢复算法

## WAL 记录结构

- 文件头：magic/version 等
- 记录头：`recordSize`、`txnId`、`type`、`checksum`
- 负载：实体变更数据（可选）

所有记录通过 CRC32 做完整性校验。

## 读取与校验

`WALManager::readRecords()` 会：

1. 校验文件头
2. 顺序读取 record header + payload
3. 校验 payload CRC32
4. 标记是否损坏

## 启动恢复路径（当前）

数据库打开时会检查 WAL 是否存在待恢复内容；当前默认行为是执行 checkpoint 清理 WAL，以保持主存储与日志状态一致。

如果你计划扩展为完整重放恢复，可从 `readRecords()` 返回结果继续构建 replay pipeline。
