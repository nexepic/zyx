# 关系遍历

## 数据组织

每个节点维护首入边/首出边，边实体维护前后指针，形成链表结构。

## 主要能力

- `getOutgoingEdges`
- `getIncomingEdges`
- `getAllConnectedEdges`
- `getConnectedTargetNodes` / `getConnectedSourceNodes`

## 写路径维护

- `linkEdge`：创建边时挂接到源/目标节点链表头
- `unlinkEdge`：删除边时修复前后链接

## 健壮性

遍历时维护 `visitedEdgeIds`，检测链表环并抛错，避免死循环。
