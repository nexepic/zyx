# 查询优化算法

## 优化框架

`Optimizer` 对逻辑计划做 fixed-point 迭代优化：

- 每轮按规则顺序应用
- 若计划不再变化则提前收敛
- 有最大迭代上限保护

## 默认规则顺序

1. `PredicateSimplificationRule`
2. `FilterPushdownRule`
3. `ProjectionPushdownRule`
4. `EnhancedIndexSelectionRule`
5. `JoinReorderRule`

顺序有意设计：先规范化谓词，再下推，再做索引与连接策略选择。

## 输出

优化结果仍是逻辑计划树，随后由 `PhysicalPlanConverter` 转为物理算子树执行。
