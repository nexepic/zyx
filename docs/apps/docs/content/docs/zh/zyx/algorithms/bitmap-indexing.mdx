# 位图索引算法

这里的“位图”主要指存储段内部的活跃槽位位图，不是独立查询引擎位图索引。

## 数据结构

`SegmentHeader` 内包含：

- `bitmap_size`
- `activity_bitmap[MAX_BITMAP_SIZE]`

每一位对应一个槽位：

- `1`：活跃
- `0`：空闲/失效

## 作用

- 快速判断实体槽是否可读
- 统计活跃数与空闲数
- 辅助碎片率计算与压缩决策

## 关键接口

- `SegmentTracker::setEntityActive(...)`
- `SegmentTracker::isEntityActive(...)`
- `SegmentTracker::countActiveEntities(...)`
- `storage::bitmap::setBit/getBit(...)`
