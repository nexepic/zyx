# 段分配算法

## 目标

- 低成本分配新段
- 复用空闲段，减少文件膨胀
- 在高碎片时触发整理和合并

## 主要组件

- `SegmentTracker`：维护 segment header 缓存、类型链表、free list
- `SpaceManager`：执行 allocate/deallocate/compact/merge/truncate

## 分配路径（简化）

1. 优先从 `freeSegments` 复用
2. 否则在文件尾扩展新段
3. 初始化 `SegmentHeader` 并挂到对应类型链表
4. 更新链头与元数据

## 回收与整理

- `deallocateSegment` 将段从类型链表摘除并标记为空闲
- `compactSegments` 按碎片比例筛选候选段
- `mergeSegments` 尝试合并低利用率段
- `truncateFile` 在条件满足时回收文件尾空间
