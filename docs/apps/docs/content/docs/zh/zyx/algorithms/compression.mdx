# 压缩算法

## 实现

当前压缩工具位于：

- `utils::zlibCompress`
- `utils::zlibDecompress`

使用 zlib 进行无损压缩/解压。

## 使用场景

- 适合较大文本或序列化负载的存储压缩
- 与 Blob/State 等大对象路径协同（具体由上层组件决定何时压缩）

## 设计取舍

- 优点：降低磁盘占用与 IO 体积
- 成本：增加 CPU 压缩/解压开销

通常建议在“体积明显大于压缩开销”时启用。

## 源码定位

- `include/graph/utils/CompressUtils.hpp`
- `src/utils/CompressUtils.cpp`
