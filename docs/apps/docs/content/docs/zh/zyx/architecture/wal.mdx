# WAL 实现

WAL 由 `WALManager` 管理，文件路径为 `<db_path>-wal`。

## 记录类型

`WALRecordType` 当前包含：

- `WAL_TXN_BEGIN`
- `WAL_TXN_COMMIT`
- `WAL_TXN_ROLLBACK`
- `WAL_ENTITY_WRITE`

每条记录含 header 与 CRC32 校验。

## 写入机制

- 内存缓冲：`writeBuffer_`
- 缓冲满时先刷盘（不 fsync）
- commit 时采用 group commit（微秒级延迟窗口）
- `sync()` 执行 flush + fsync

## checkpoint

满足条件时执行 checkpoint：

- 刷新并 fsync
- 重建 WAL 文件并写入新 header
- 重置写偏移

## 关于恢复

当前启动阶段会检查 WAL 是否有未消费数据，并进行 checkpoint 清理。恢复路径仍在持续演进中，文档应以 `Database::ensureWALAndTransactionManager()` 的当前实现为准。
