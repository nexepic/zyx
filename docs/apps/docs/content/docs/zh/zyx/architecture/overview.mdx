# 架构概述

ZYX 是可嵌入式图数据库引擎，核心是单进程内库调用模型，不依赖独立服务进程。

## 分层结构

1. 应用层：CLI、benchmark、宿主应用
2. API 层：C++ API（`zyx.hpp`）与 C API（`zyx_c_api.h`）
3. 查询层：Parser -> Logical Plan -> Optimizer -> Physical Operators
4. 存储层：FileStorage、DataManager、IndexManager、WALManager
5. 核心层：Database、Transaction、TransactionManager

## 关键对象

- `graph::Database`
  - 负责存储打开、配置加载、查询引擎/线程池惰性初始化
- `graph::query::QueryEngine`
  - 负责语句解析、计划生成与执行
- `graph::storage::FileStorage`
  - 负责文件结构、段管理、持久化刷新
- `graph::TransactionManager`
  - 负责单写事务与只读快照事务

## 主路径

### 查询路径

`execute(cypher)` -> parser -> logical plan -> optimizer -> physical plan -> operators -> `QueryResult`

### 写入路径

operator 写入 DataManager -> 脏数据进入 PersistenceManager -> commit 时 WAL + save/flush
