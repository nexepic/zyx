# 优化策略

性能优化主要发生在三层：逻辑优化、物理执行策略、存储读写路径。

## 逻辑优化规则

默认规则顺序：

1. 谓词简化（PredicateSimplification）
2. 过滤下推（FilterPushdown）
3. 投影下推（ProjectionPushdown）
4. 增强索引选择（EnhancedIndexSelection）
5. Join 重排（JoinReorder）

规则采用 fixed-point 迭代直到收敛或达到迭代上限。

## 扫描策略选择

`NodeScan` 会结合统计信息与索引可用性，在以下策略间选择：

- 全量扫描
- 标签/属性索引扫描
- 范围扫描
- 复合索引扫描

未被索引吸收的条件会转为 residual filter。

## 执行层优化点

- 多标签匹配使用 residual 多标签过滤
- 并发路径下支持批量加载实体与并行扫描
- 向量查询可走专用 vector operator
