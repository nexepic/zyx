# 事务管理

事务由 `TransactionManager` 管理，`Transaction` 提供外部句柄。

## 状态机

`Transaction::TxnState`：

- `TXN_ACTIVE`
- `TXN_COMMITTED`
- `TXN_ROLLED_BACK`

`Transaction` 为 move-only；若对象析构时仍为 active，会自动回滚。

## 并发策略

- 写事务：单写（`std::shared_mutex` 独占锁）
- 只读事务：多读（共享锁 + `CommittedSnapshot`）

这意味着：

- 只读事务可并发运行
- 同一时刻只有一个写事务可以提交修改

## 提交流程（写事务）

1. 写 WAL commit 记录并 `sync`
2. `FileStorage::save()` 持久化主存储
3. 达到阈值时触发 WAL checkpoint
4. 发布新的 committed snapshot
5. 清理事务上下文并释放写锁

## 回滚流程

1. DataManager 回滚当前事务脏修改
2. 记录 WAL rollback
3. checkpoint 清理 WAL
4. 清理事务上下文并释放锁
