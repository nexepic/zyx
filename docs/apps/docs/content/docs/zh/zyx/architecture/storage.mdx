# 存储系统

存储层由 `FileStorage` 统筹，`DataManager` 负责实体读写，`SegmentTracker/SpaceManager` 负责空间管理。

## 核心组件职责

- `FileStorage`
  - 打开/关闭文件
  - 组件初始化与 flush/save 协调
- `DataManager`
  - Node/Edge/Property/Blob/Index/State 的统一入口
  - 脏数据管理与事务上下文
- `PersistenceManager`
  - 维护各实体类型 dirty registry
  - 触发自动 flush
- `SegmentTracker`
  - 跟踪 segment header 与链表关系
- `IDAllocator`
  - 管理实体 ID 分配与恢复

## 存储对象类型

系统内统一管理六类实体：

- Node
- Edge
- Property
- Blob
- Index
- State

各类型使用各自的 segment chain 与索引结构。

## 读写路径（简化）

1. 查询算子调用 DataManager
2. DataManager 优先走事务上下文/脏数据视图
3. 命中失败则回落到缓存或磁盘读取
4. 写入进入 dirty registry，提交时统一持久化

## 校验与运维支持

- 完整性检查：`FileStorage::verifyIntegrity()`
- 调试检查器：`DatabaseInspector`
- REPL `debug` 命令可检查 nodes/edges/props/blobs/indexes/states
