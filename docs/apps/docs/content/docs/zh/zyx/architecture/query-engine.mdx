# 查询引擎

当前查询引擎分为 4 段：解析、逻辑计划、优化、物理执行。

## 1. 解析层

- 语法：`src/query/parser/cypher/generated/CypherParser.g4`
- 入口：`CypherParserImpl` + `CypherToPlanVisitor`
- 子句处理：`clauses/*Handler.cpp`

当前已覆盖的核心语法块：

- 读：`MATCH`、`OPTIONAL MATCH`、`UNWIND`、`CALL`
- 写：`CREATE`、`MERGE`、`SET`、`REMOVE`、`DELETE/DETACH DELETE`
- 结果：`RETURN`、`ORDER BY`、`SKIP`、`LIMIT`、`DISTINCT`
- 组合：`WITH`、`UNION/UNION ALL`
- 管理：索引/约束 DDL、`SHOW INDEX/CONSTRAINT`
- 事务语句：`BEGIN/COMMIT/ROLLBACK`

## 2. 逻辑计划层

统一抽象为 `LogicalOperator` 树（见 `include/graph/query/logical/LogicalOperator.hpp`）。

- 读操作：`LOP_NODE_SCAN`、`LOP_FILTER`、`LOP_PROJECT`、`LOP_JOIN`...
- 写操作：`LOP_CREATE_NODE`、`LOP_MERGE_EDGE`、`LOP_SET`...
- 管理操作：`LOP_CREATE_INDEX`、`LOP_CREATE_CONSTRAINT`...

## 3. 优化层

默认规则（`src/query/optimizer/Optimizer.cpp`）：

1. `PredicateSimplificationRule`
2. `FilterPushdownRule`
3. `ProjectionPushdownRule`
4. `EnhancedIndexSelectionRule`
5. `JoinReorderRule`

## 4. 物理执行层

`PhysicalPlanConverter` 把逻辑计划映射为 `PhysicalOperator` 树，再由 `QueryExecutor` 执行。

常见物理算子：

- 扫描与过滤：`NodeScanOperator`、`FilterOperator`
- 图扩展：`TraversalOperator`、`VarLengthTraversalOperator`、`OptionalMatchOperator`
- 聚合与投影：`AggregateOperator`、`ProjectOperator`、`SortOperator`
- 写入与管理：`Create*`、`Merge*`、`Set/Remove/Delete`、`CreateIndex/Constraint`

## 源码定位

- `src/query/parser/cypher/CypherToPlanVisitor.cpp`
- `src/query/planner/PhysicalPlanConverter.cpp`
- `include/graph/query/optimizer/Optimizer.hpp`
