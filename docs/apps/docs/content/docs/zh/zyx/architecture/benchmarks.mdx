# 基准测试

ZYX 提供独立 benchmark 应用（`apps/benchmark`），覆盖插入、查询、并发、向量检索与优化器场景。

## 覆盖场景

| 分类 | 测试内容 |
|------|----------|
| **Insert** | Cypher 与 Native API 写入路径，有/无索引 |
| **Query** | 属性索引查找、标签扫描、全扫描 |
| **Algo** | 最短路径等图算法 |
| **Vector** | 向量插入与 ANN（近似最近邻）查询 |
| **Concurrency** | 多线程读、写、排序、向量检索 |
| **Optimizer** | 优化规则效果并排对比 |

## 运行方式

构建 benchmark 二进制文件后直接运行：

```bash
./buildDir/apps/benchmark/benchmark
```

benchmark 框架（`BenchmarkFramework`）通过 `BenchmarkConfig` 控制数据集大小、线程数、预热迭代和测量时长。

## 结果解读

阅读 benchmark 输出时请注意：

1. **固定硬件和编译选项** — 结果因 CPU、存储设备和优化级别而异。对比时始终使用相同的 `-O3` release 构建。
2. **区分冷启动和稳态** — 首次运行会填充页缓存和创建 WAL；后续运行受益于热缓存。两者都应报告。
3. **使用 `PROFILE` 和 `EXPLAIN`** — 将 benchmark 数值与查询计划分析结合，理解查询快或慢的原因。
4. **检查索引命中率** — 查询可能不是因为引擎慢，而是缺少预期索引。
5. **跨版本对比时保持完全相同的配置** — 同一编译器、同一数据分布、同一硬件。
