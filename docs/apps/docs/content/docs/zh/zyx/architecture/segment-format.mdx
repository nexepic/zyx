# 段格式

ZYX 使用自定义的基于段的存储格式，提供高效的空间管理、快速访问和 ACID 合规性。

## 概述

数据库文件被划分为固定大小的段，每个段独立管理。这种设计实现了：
- **高效空间分配**：基于位图的空间跟踪
- **快速访问**：直接段偏移计算
- **并发操作**：段级锁定
- **易于恢复**：段级校验和和元数据

### 数据库文件结构

#### 整体布局



#### 文件头详情

| 字段 | 大小 | 描述 | 示例值 |
|-------|------|-------------|---------------|
| 魔术数字 | 8 字节 | 文件标识符: "METRIXXZ" | - |
| 版本 | 4 字节 | 格式版本号 | 1 |
| 页大小 | 4 字节 | 系统页大小（通常为 4096） | 4096 |
| 段大小 | 8 字节 | 每个段的大小（默认：4MB） | 4194304 |
| 段数量 | 8 字节 | 段的总数 | N |
| WAL 偏移 | 8 字节 | 预写日志的偏移量 | - |

#### 段结构详情



| 组件 | 大小 | 描述 |
|-----------|------|-------------|
| **段头** | 64 字节 | 段元数据和 ID |
| **空闲空间位图** | 8 KB | 跟踪 64 字节块的分配 |
| **数据页** | ~4 MB | 实际实体存储 |

#### 预写日志 (WAL)

| 组件 | 描述 |
|-----------|-------------|
| 日志条目 | 事务日志记录 |
| 检查点元数据 | 检查点信息 |
| 事务记录 | 事务状态数据 |

### 段层次结构



**段组织结构：**
- 每个段：默认 4MB 大小
- 段 0-N：数据存储段
- 预写日志：确保持久性

## 文件头

文件头包含数据库范围的元数据。

## 段结构

每个段具有固定大小（默认：1MB），包含：
- 段头：段元数据
- 实体数据：实际存储的数据
- 自由空间位图：空间分配跟踪

## 实体存储

### 数据页布局



#### 页头详情

| 字段 | 类型 | 描述 | 示例值 |
|-------|------|-------------|---------------|
| pageId | uint32_t | 页面唯一标识符 | 42 |
| segmentId | uint32_t | 所属段的 ID | 0 |
| entityType | uint32_t | 实体类型 (节点/边/属性) | NODE |
| entityCount | uint32_t | 存储的实体数量 | 45 |
| freeSpace | uint32_t | 可用空闲空间（字节） | 1024 |
| checksum | uint32_t | CRC32 校验和用于验证 | 0xABCD1234 |

#### 实体槽表

槽表跟踪页内实体的分配情况（每页 32 个槽）：

| 槽 | 状态 | 偏移 | 大小 |
|------|--------|--------|------|
| 0 | 已用 | 0 | 28 |
| 1 | 空闲 | - | - |
| 2 | 已用 | 64 | 36 |
| 3 | 已用 | 92 | 44 |
| ... | ... | ... | ... |
| 31 | 已用 | 3850 | 128 |

#### 实体数据区

实体连续存储，大小可变。空闲空间穿插在实体之间。

| 实体 | 偏移 | 大小 | 详情 |
|--------|--------|------|---------|
| 实体 0 | 0 | 28 | `id: 0x0001, label: "Person", properties: {"name": "Alice", "age": 30}` |
| 实体 2 | 64 | 36 | `id: 0x0003, label: "User", properties: {"email": "bob@ex.com"}, outgoing: [0x0005, 0x0008]` |
| ... | ... | ... | ... |

实体在段内存储，包含：
- 实体头：元数据和校验和
- 实体数据：实际属性和关系

## 自由空间管理

使用位图分配器跟踪段内的可用空间，每个位对应一个固定大小的块（64 字节）。

## 最佳实践

1. **批量操作**：分组多个实体操作
2. **使用适当的段大小**：根据工作负载调整
3. **监控碎片**：定期回收空间
4. **策略性压缩**：压缩写入频率低的段
5. **验证校验和**：定期检查数据完整性
6. **优先顺序访问**：比随机访问更高效
