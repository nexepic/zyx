# 段格式

底层结构定义在 `StorageHeaders.hpp`。

## 文件级结构

- `FileHeader`：魔数、版本、各实体链表头、max id、CRC
- 魔数：`<ZYX-DB>`
- 版本字段：`version`

## 段级结构

`SegmentHeader` 记录：

- 段偏移与前后指针
- `start_id`、`capacity`、`used`、`inactive_count`
- `data_type`
- `activity_bitmap`
- `segment_crc`
- `needs_compaction` / `is_dirty`

## 活跃位图

- bitmap 记录实体槽位是否活跃
- 支持快速计算 active/free 数量
- 与 `inactive_count` 一起反映碎片化程度

## 为什么文档不写死容量数字

段尺寸、每段实体容量等常量由代码常量推导（如 `SEGMENT_SIZE`、`NODES_PER_SEGMENT`）。为避免文档滞后，推荐直接查看头文件常量，而不是依赖静态文档表格。

- `include/graph/storage/SegmentTracker.hpp`
