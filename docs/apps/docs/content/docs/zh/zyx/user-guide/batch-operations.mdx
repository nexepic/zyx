# 批处理操作

## 核心原则

1. 分批提交，避免单事务过大。
2. 优先参数化，减少解析与字符串拼接开销。
3. 必要时走 API 的原生批量接口，而不是纯 Cypher 循环。

## 方案 A：Cypher + 分批事务

```cypher
BEGIN;
UNWIND [
  {name: 'u1', age: 20},
  {name: 'u2', age: 21}
] AS row
CREATE (:User {name: row.name, age: row.age});
COMMIT;
```

## 方案 B：脚本批量执行

```bash
./buildDir/apps/cli/zyx database exec ./demo.graph ./batch.cypher
```

## 方案 C：C++ 原生批量接口

对超大导入任务，优先考虑：

- `Database::createNodes(...)`
- `Database::createNodeRetId(...)`
- `Database::createEdgeById(...)`

这样可以减少语法解析与匹配成本。
