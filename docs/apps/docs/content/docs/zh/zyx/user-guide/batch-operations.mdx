# 批处理操作

## 策略选择

```mermaid
flowchart TD
  A[需要批量写入] --> B{数据来源}
  B -->|内存行数据| C[UNWIND + 事务]
  B -->|CSV 文件| D[LOAD CSV 或 CLI import]
  B -->|应用服务| E[C++/C API 原生批量方法]
  C --> F{数据量是否很大}
  F -->|是| G[CALL 子查询 IN TRANSACTIONS]
  F -->|否| H[单个显式事务]
```

## 方案 A：UNWIND + 显式事务

```cypher
BEGIN;
UNWIND [
  {name: 'u1', age: 20},
  {name: 'u2', age: 21}
] AS row
CREATE (:User {name: row.name, age: row.age});
COMMIT;
```

## 方案 B：子查询事务分批

```cypher
UNWIND $rows AS row
CALL {
  WITH row
  MERGE (:User {name: row.name})
} IN TRANSACTIONS OF 1000 ROWS
RETURN count(*) AS processed;
```

## 方案 C：脚本模式执行

```bash
./buildDir/apps/cli/zyx database exec ./demo.graph ./batch.cypher
```

适合重复执行的运维任务与 CI 初始化数据。

## 方案 D：原生批量 API（服务端）

在极限吞吐场景优先使用：

- `Database::createNodes(...)`
- `Database::createNodeRetId(...)`
- `Database::createEdgeById(...)`

## 运维检查清单

- 固定批大小与提交节奏。
- 记录成功/失败批次范围，便于断点恢复。
- 对可重放流程设计幂等键（必要时使用 `MERGE`）。
- 每个大批次后执行一致性校验查询。
