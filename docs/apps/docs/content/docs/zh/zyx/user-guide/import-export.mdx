# 导入与导出

高效地在 ZYX 中导入和导出数据对于数据迁移、备份和系统集成至关重要。本指南涵盖导入和导出操作。

## 数据导入

### CSV 导入

#### 基本 CSV 导入

```cypher
// 从 CSV 文件导入
LOAD CSV FROM 'file:///path/to/users.csv' AS row
CREATE (u:User {
  id: toInteger(row[0]),
  name: row[1],
  email: row[2]
})
```

#### 批量 CSV 导入

```cypher
// 批量导入以提高性能
USING PERIODIC COMMIT 1000
LOAD CSV WITH HEADERS FROM 'file:///path/to/users.csv' AS row
CREATE (u:User {
  id: toInteger(row.id),
  name: row.name,
  email: row.email
})
```

### JSON 导入

```cypher
// 导入行分隔的 JSON
CALL apoc.loadJson('file:///path/to/users.json')
YIELD value
CREATE (u:User {
  id: value.id,
  name: value.name,
  email: value.email
})
```

## 数据导出

### 导出到 CSV

```cypher
// 导出所有节点到 CSV
MATCH (n:User)
RETURN n.id AS id, n.name AS name, n.email AS email
CALL apoc.export.csv.query(
  'MATCH (n:User) RETURN n.id AS id, n.name AS name, n.email AS email',
  'users_export.csv',
  {}
)
```

## 导入策略

### 增量导入

```cypher
// 仅导入新记录
LOAD CSV WITH HEADERS FROM 'file:///path/to/new_users.csv' AS row
MERGE (u:User {id: toInteger(row.id)})
ON CREATE SET
  u.name = row.name,
  u.email = row.email,
  u.createdAt = timestamp()
ON MATCH SET
  u.lastUpdated = timestamp()
```

### Upsert 操作

```cypher
// 如果存在则更新，否则创建
LOAD CSV WITH HEADERS FROM 'file:///path/to/users.csv' AS row
MERGE (u:User {email: row.email})
SET u.name = row.name,
    u.age = toInteger(row.age)
```

## 处理大数据集

### 流式处理

```cypher
// 分块处理大文件
USING PERIODIC COMMIT 5000
LOAD CSV WITH HEADERS FROM 'file:///path/to/large_file.csv' AS row
CREATE (n:Node {
  id: toInteger(row.id),
  data: row.data
})
```

## 最佳实践

1. **使用定期提交**：对于大型导入（>100K 行）
2. **验证首先**：导入前检查数据质量
3. **备份现有数据**：批量操作前
4. **使用事务**：分组相关操作
5. **监控资源**：观察内存和磁盘使用情况
6. **测试样本**：首先导入小样本
7. **记录映射**：跟踪字段映射
8. **处理错误**：为无效数据制定计划
