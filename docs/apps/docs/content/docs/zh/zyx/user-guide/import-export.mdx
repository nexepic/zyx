# 导入与导出

## 当前能力边界

当前实现中，`LOAD CSV` / APOC 导入导出并不是内置能力。推荐使用以下路径。

## 导入路径

### 1) CLI 脚本导入

- 将数据转换为 Cypher 语句文件
- 使用 `database exec` 执行

### 2) C++/C API 导入

- 参数化 `execute`
- 或使用原生写入 API（`createNodes` / `createEdgeById`）

## 导出路径

### 1) 查询结果导出

- 使用 API 迭代 `Result`/`ZYXResult_T`
- 由应用层写出 CSV/JSON/Parquet

### 2) 物理备份

- 先执行 `save`（或 API `save()`）并关闭数据库
- 直接复制数据库文件

## 备份建议

- 备份前确保没有活跃写事务
- 使用定期快照 + 增量导出组合
- 恢复演练至少定期执行一次
