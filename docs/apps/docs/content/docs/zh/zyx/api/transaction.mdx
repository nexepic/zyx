# 事务模型

ZYX 当前同时支持三种事务入口。

## 1. API 显式事务

```cpp
auto tx = db.beginTransaction();
auto r = tx.execute("CREATE (n:User {name: 'Alice'})");
if (r.isSuccess()) tx.commit(); else tx.rollback();
```

特点：

- 边界最明确，适合业务层控制多条语句原子提交。
- `Transaction` 析构时会自动回滚未提交事务。

## 2. Cypher 事务语句

在 `Database::execute(...)` 中可直接执行：

- `BEGIN`
- `COMMIT`
- `ROLLBACK`

特点：

- 适合 CLI/脚本场景。
- 不支持嵌套事务。

## 3. 隐式事务

当没有显式事务时，`Database::execute(...)` 会按语句类型处理：

- 读查询：走只读快速路径（不包隐式写事务）
- 写查询：自动开启并提交隐式事务

## 当前并发模型（实现口径）

- 写事务：单写者（`TransactionManager` 使用独占锁）
- 只读事务：可并发，基于快照对象读取
