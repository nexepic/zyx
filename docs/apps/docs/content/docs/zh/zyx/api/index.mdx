# API 参考

ZYX 暴露两种嵌入式接口 — 现代 C++ API 和用于 FFI 绑定的 C 兼容 API。

## 如何选择？

- **C++ API**（`include/zyx/zyx.hpp`）— C++20 开发首选。类型安全的 Value、RAII 事务管理、流式 `QueryBuilder`。
- **Python API**（`pip install zyxdb`）— Python 开发首选。提供 Pythonic 接口，支持上下文管理器、迭代器和 pandas 集成。
- **C API**（`include/zyx/zyx_c_api.h`）— 构建 Rust、Go、Node.js 或 WASM 绑定时使用。不透明句柄，显式生命周期管理。

## 设计约定

- 查询执行统一返回 `Result` / `ZYXResult_T*`，由调用方显式迭代结果集。
- 事务支持三种形式：
  - **API 级显式事务** — `beginTransaction()` / `zyx_begin_transaction()`
  - **只读事务** — `beginReadOnlyTransaction()` / `zyx_begin_read_only_transaction()`
  - **Cypher 事务语句** — 在 `execute()` 中使用 `BEGIN` / `COMMIT` / `ROLLBACK`
- 无显式事务时，`execute()` 对写查询自动提交，对读查询走只读快速路径。
