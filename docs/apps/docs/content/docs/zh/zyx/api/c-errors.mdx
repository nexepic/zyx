# C API 错误处理

C API 使用**返回值 + 错误字符串**模式 — 无错误码枚举。

## 1. API 级错误

当函数返回 `NULL` 或 `false` 时，调用 `zyx_get_last_error()` 获取错误消息：

```c
ZYXDB_T* db = zyx_open("./mydb");
if (!db) {
    printf("打开数据库失败: %s\n", zyx_get_last_error());
    return 1;
}

ZYXTxn_T* tx = zyx_begin_transaction(db);
if (!tx) {
    printf("开始事务失败: %s\n", zyx_get_last_error());
    zyx_close(db);
    return 1;
}
```

`zyx_get_last_error()` 读取**线程本地存储** — 每个线程有独立的最近错误。多线程并发调用安全。

## 2. Result 级错误

查询执行返回的 `ZYXResult_T*` 可能包含查询级错误：

```c
ZYXResult_T* res = zyx_execute(db, "INVALID CYPHER");
if (!zyx_result_is_success(res)) {
    printf("查询失败: %s\n", zyx_result_get_error(res));
    zyx_result_close(res);
    return 1;
}
zyx_result_close(res);
```

在迭代结果前始终检查 `zyx_result_is_success()`。

## 推荐错误检查模式

```c
// 1. 检查返回值（NULL / false）
// 2. 失败时读取 zyx_get_last_error()
// 3. 对查询结果再检查 zyx_result_is_success()
// 4. 查询失败时读取 zyx_result_get_error()

ZYXDB_T* db = zyx_open("./mydb");
if (!db) { /* 检查 zyx_get_last_error() */ }

ZYXResult_T* res = zyx_execute(db, "MATCH (n) RETURN n");
if (!res) { /* 分配失败 — 检查 zyx_get_last_error() */ }

if (!zyx_result_is_success(res)) {
    printf("查询错误: %s\n", zyx_result_get_error(res));
    zyx_result_close(res);
    zyx_close(db);
    return 1;
}

// 安全迭代
while (zyx_result_next(res)) {
    // ...
}

zyx_result_close(res);
zyx_close(db);
```

## 错误字符串生命周期

| 函数 | 有效期至 |
|------|---------|
| `zyx_get_last_error()` | 同线程上的下一次 ZYX API 调用 |
| `zyx_result_get_error()` | `zyx_result_close()` |

如需超出上述有效期保留字符串，请立即复制。
