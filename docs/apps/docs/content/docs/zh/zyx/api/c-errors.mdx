# C API 错误处理

C API 当前不使用独立错误码枚举，采用“返回值 + 错误字符串”模式。

## 1. API 调用失败

当函数返回 `NULL` 或 `false` 时，调用：

- `zyx_get_last_error()`

该错误消息是线程本地存储（thread-local）。

## 2. 查询结果失败

`zyx_execute*` 返回 `ZYXResult_T*` 后，仍建议检查：

- `zyx_result_is_success(res)`
- `zyx_result_get_error(res)`

## 推荐处理流程

1. 检查函数返回值（是否 `NULL`/`false`）
2. 若失败，读取 `zyx_get_last_error()`
3. 对查询结果再检查 `zyx_result_is_success()`
4. 读取 `zyx_result_get_error()` 作为业务层错误信息
