# Node.js API 参考

包名：`zyxdb`（基于 N-API 的原生插件）

## 安装

```bash
npm install zyxdb
```

平台特定的预编译二进制文件通过可选依赖自动安装（`@nexepic/zyxdb-linux-x64-gnu`、`@nexepic/zyxdb-darwin-arm64`、`@nexepic/zyxdb-win32-x64-msvc`）。

## Database

`Database` 是主入口。所有 I/O 方法均为异步。

### 构造与生命周期

```javascript
const { Database } = require('zyxdb');

// 创建并打开
const db = new Database('/path/to/db');
await db.open();

// 使用数据库...

// 使用完毕后关闭
await db.close();
```

| 方法 | 签名 | 描述 |
|------|------|------|
| 构造函数 | `new Database(path: string)` | 创建数据库实例。 |
| `open` | `open(): Promise<void>` | 打开数据库（不存在时自动创建）。 |
| `close` | `close(): Promise<void>` | 刷盘并关闭数据库。 |
| `save` | `save(): Promise<void>` | 将脏数据刷到磁盘，不关闭数据库。 |

### 查询执行

```javascript
// 自动提交查询
const result = await db.execute('CREATE (n:Person {name: "Alice", age: 30})');

// 参数化查询
const result = await db.execute(
  'MATCH (n:Person) WHERE n.name = $name RETURN n.age AS age',
  { name: 'Alice' }
);

// 遍历结果
for (const row of result) {
  console.log(row.age);       // 通过列名访问（Proxy）
  console.log(row.get(0));    // 通过列索引访问
}
```

| 方法 | 签名 | 描述 |
|------|------|------|
| `execute` | `execute(cypher: string, params?: object): Promise<Result>` | 执行 Cypher 查询，支持可选参数。 |

### 事务

```javascript
// 写事务
const tx = await db.beginTransaction();
try {
  await tx.execute('CREATE (n:Person {name: "Alice"})');
  await tx.execute('CREATE (n:Person {name: "Bob"})');
  await tx.commit();
} catch (e) {
  await tx.rollback();
  throw e;
}

// 只读事务（快照隔离）
const tx = await db.beginReadOnlyTransaction();
const result = await tx.execute('MATCH (n:Person) RETURN n.name AS name');
await tx.commit();
```

| 方法 | 签名 | 描述 |
|------|------|------|
| `beginTransaction` | `beginTransaction(): Promise<Transaction>` | 开始读写事务。 |
| `beginReadOnlyTransaction` | `beginReadOnlyTransaction(): Promise<Transaction>` | 开始只读事务（快照隔离）。 |
| `hasActiveTransaction` | `get hasActiveTransaction: boolean` | 是否存在活动事务。 |

### 高性能写入 API

绕过 Cypher 解析器，获得最大写入吞吐。所有创建方法返回 ID。

```javascript
// 创建节点（返回节点 ID）
const aliceId = await db.createNode('Person', { name: 'Alice', age: 30 });

// 创建多标签节点
const empId = await db.createNode(['Person', 'Employee'], { name: 'Bob' });

// 批量创建节点（返回 ID 数组）
const ids = await db.createNodes('Person', [
  { name: 'Alice', age: 30 },
  { name: 'Bob', age: 25 },
  { name: 'Charlie', age: 35 },
]);

// 创建边（返回边 ID）
const edgeId = await db.createEdge(aliceId, empId, 'KNOWS', { since: 2020 });

// 批量创建边（返回 ID 数组）
const edgeIds = await db.createEdges('FOLLOWS', [
  [id1, id2, { weight: 1.0 }],
  [id2, id3, null],           // 无属性
]);
```

| 方法 | 签名 | 描述 |
|------|------|------|
| `createNode` | `createNode(label: string \| string[], props?: object): Promise<number>` | 创建节点并返回其内部 ID。 |
| `createNodes` | `createNodes(label: string, propsList: object[]): Promise<number[]>` | 批量创建同标签节点。 |
| `createEdge` | `createEdge(srcId: number, dstId: number, edgeType: string, props?: object): Promise<number>` | 通过节点 ID 创建边。 |
| `createEdges` | `createEdges(edgeType: string, edges: Array): Promise<number[]>` | 批量创建边。每个元素为 `[srcId, dstId, props?]`。 |

### 图算法

```javascript
// 最短路径 — 返回节点对象数组
const path = await db.getShortestPath(startId, endId, 15);
for (const node of path) {
  console.log(node.id, node.labels, node.properties);
}
```

| 方法 | 签名 | 描述 |
|------|------|------|
| `getShortestPath` | `getShortestPath(startId: number, endId: number, maxDepth?: number): Promise<object[]>` | 查找最短路径，返回路径上的节点对象数组。 |

## Transaction

`Transaction` 提供 ACID 事务语义。必须显式提交或回滚。

```javascript
const tx = await db.beginTransaction();
try {
  await tx.execute('CREATE (n:Person {name: "Alice"})');
  await tx.commit();
} catch (e) {
  await tx.rollback();
  throw e;
}
```

| 方法 | 签名 | 描述 |
|------|------|------|
| `execute` | `execute(cypher: string, params?: object): Promise<Result>` | 在此事务中执行 Cypher 查询。 |
| `commit` | `commit(): Promise<void>` | 提交事务。 |
| `rollback` | `rollback(): Promise<void>` | 回滚事务。 |
| `isActive` | `get isActive: boolean` | 事务是否仍然活跃。 |
| `isReadOnly` | `get isReadOnly: boolean` | 是否为只读事务。 |

对已关闭的事务调用 `execute`、`commit` 或 `rollback` 会抛出 `Error`。

## Result

`Result` 封装查询输出，支持迭代和便捷方法。

```javascript
const result = await db.execute('MATCH (n:Person) RETURN n.name AS name, n.age AS age');

// 以 Record 对象迭代
for (const row of result) {
  console.log(row.name, row.age);
}

// 便捷方法
const all = result.fetchAll();
const one = result.single();        // 不是恰好 1 行时抛异常
const val = result.scalar();        // 第一行第一列
const data = result.data();         // 普通对象数组
```

| 方法 / 属性 | 签名 | 描述 |
|-------------|------|------|
| `columns` | `get columns: string[]` | 结果集的列名。 |
| `isSuccess` | `get isSuccess: boolean` | 查询是否执行成功。 |
| `error` | `get error: string \| null` | 错误信息，成功时为 `null`。 |
| `duration` | `get duration: number` | 查询执行耗时（毫秒）。 |
| `records` | `get records: Record[]` | 所有行作为 Record 对象。 |
| `[Symbol.iterator]` | `*[Symbol.iterator](): Iterator<Record>` | 以 Record 对象迭代。 |
| `fetchAll` | `fetchAll(): Record[]` | 获取所有行作为 Record 对象。 |
| `single` | `single(strict?: boolean): Record` | 返回唯一结果行。`strict`（默认）为 true 且行数 != 1 时抛异常。 |
| `scalar` | `scalar(): any` | 返回第一行第一列的值。 |
| `data` | `data(): object[]` | 返回所有行作为普通对象。 |

## Record

`Record` 表示一行结果。支持通过 Proxy 直接属性访问（如 `row.name`）和显式 `get()` 方法。

```javascript
for (const row of result) {
  // 直接属性访问（Proxy）
  console.log(row.name);

  // 按索引访问
  console.log(row.get(0));

  // 元数据
  const keys = row.keys();      // ['name', 'age']
  const vals = row.values();    // ['Alice', 30]
  const obj = row.data();       // { name: 'Alice', age: 30 }
}
```

| 方法 | 签名 | 描述 |
|------|------|------|
| `get` | `get(key: string \| number): any` | 按列名或索引访问。 |
| `keys` | `keys(): string[]` | 返回列名。 |
| `values` | `values(): any[]` | 按列顺序返回所有值。 |
| `data` | `data(): object` | 以普通对象形式返回该行。 |
| `length` | `get length: number` | 列数。 |

## 错误处理

```javascript
const { Database } = require('zyxdb');

// 查询错误捕获在 Result 中
const result = await db.execute('INVALID SYNTAX');
if (!result.isSuccess) {
  console.error(`查询失败: ${result.error}`);
}

// 原生错误抛出标准 Error
try {
  await db.createEdge(999999, 999998, 'LINK');
} catch (e) {
  console.error(`数据库错误: ${e.message}`);
}
```

查询解析/执行错误捕获在 `Result.error` 中，而非以异常方式抛出。

## 完整示例

```javascript
const { Database } = require('zyxdb');

async function main() {
  const db = new Database('/tmp/social.zyx');
  await db.open();

  // 创建节点
  const alice = await db.createNode('Person', { name: 'Alice', age: 30 });
  const bob = await db.createNode('Person', { name: 'Bob', age: 25 });

  // 创建边
  await db.createEdge(alice, bob, 'KNOWS', { since: 2020 });

  // 查询
  const result = await db.execute(
    'MATCH (a:Person)-[r:KNOWS]->(b:Person) ' +
    'RETURN a.name AS src, b.name AS dst, r.since AS since'
  );
  for (const row of result) {
    console.log(`${row.src} knows ${row.dst} since ${row.since}`);
  }

  // 事务
  const tx = await db.beginTransaction();
  try {
    await tx.execute("CREATE (n:Company {name: 'ZYX'})");
    await tx.execute(
      "MATCH (p:Person {name: 'Alice'}), (c:Company {name: 'ZYX'}) " +
      "CREATE (p)-[:WORKS_AT]->(c)"
    );
    await tx.commit();
  } catch (e) {
    await tx.rollback();
    throw e;
  }

  // 标量查询
  const count = (await db.execute('MATCH (n) RETURN count(n) AS c')).scalar();
  console.log(`总节点数: ${count}`);

  await db.close();
}

main().catch(console.error);
```
