# 错误处理

当前 C++ API 未暴露模板式“异常类层次树”。应按下面两类处理。

## 1. 直接抛异常的方法

典型包括：

- `Database::open()` / `openIfExists()` / `beginTransaction()`
- `Transaction::commit()` / `rollback()`

建议在调用边界统一捕获 `std::exception`。

## 2. 返回失败 Result 的方法

典型包括：

- `Database::execute(...)`
- `Transaction::execute(...)`

这两类方法内部会捕获执行异常，并把错误写入 `Result`：

- `isSuccess() == false`
- `getError()` 返回错误文本

## 推荐模式

```cpp
auto res = db.execute("MATCH (n) RETURN n");
if (!res.isSuccess()) {
  // 统一处理查询失败
  auto msg = res.getError();
}
```

## C++/C 混合绑定建议

如果上层还会提供 C API，建议在一层 Adapter 中统一映射：

- C++ 异常 -> C 层错误字符串
- 失败 Result -> 统一错误码或错误消息
