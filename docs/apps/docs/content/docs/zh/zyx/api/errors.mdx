# 错误处理

ZYX 根据操作类型使用两种互补的错误报告机制。

## 1. 异常（生命周期操作）

无法返回 `Result` 的方法在失败时抛出 `std::exception`（或其子类）：

| 方法 | 失败场景 |
|------|---------|
| `Database::open()` | 文件 I/O 错误、文件头损坏 |
| `Database::openIfExists()` | 文件不存在、文件头损坏 |
| `Transaction::commit()` | WAL 同步失败、存储刷新失败 |
| `Transaction::rollback()` | Undo 过程中存储 I/O 错误 |

```cpp
try {
    zyx::Database db("/path/to/db");
    auto tx = db.beginTransaction();
    tx.execute("CREATE (n:Person {name: 'Alice'})");
    tx.commit();
} catch (const std::exception& e) {
    std::cerr << "错误: " << e.what() << std::endl;
}
```

在应用边界统一捕获 `std::exception`。ZYX 异常携带描述性的 `what()` 消息。

## 2. 失败 Result（查询执行）

`Database::execute()` 和 `Transaction::execute()` **不会为查询级错误抛异常**。错误被捕获在 `Result` 中：

```cpp
auto result = db.execute("INVALID CYPHER");
if (!result.isSuccess()) {
    std::cerr << "查询失败: " << result.getError() << std::endl;
}
```

| Result 方法 | 说明 |
|-------------|------|
| `isSuccess()` | 查询失败时返回 `false` |
| `getError()` | 返回错误信息字符串 |

常见查询失败原因：

- **解析错误** — 无效的 Cypher 语法
- **执行错误** — 算子执行过程中的运行时错误
- **参数错误** — 缺少参数或类型不匹配

## 推荐模式

```cpp
zyx::Database db("/path/to/db");

auto tx = db.beginTransaction();
try {
    auto result = tx.execute(
        "CREATE (n:Person {name: $name})", {{"name", std::string("Alice")}});

    if (!result.isSuccess()) {
        // 查询级错误 — 无异常抛出
        std::cerr << "查询错误: " << result.getError() << std::endl;
        // tx 析构时自动回滚
        return;
    }

    tx.commit();
} catch (const std::exception& e) {
    // 生命周期错误（WAL 同步、存储失败等）
    std::cerr << "系统错误: " << e.what() << std::endl;
}
```
