# WAL 实现

ZYX 使用写前日志（WAL）来确保 ACID 事务持久性并支持崩溃恢复。

## 概述

写前日志在将更改应用到主数据库文件之前记录所有更改。这确保了：
- **原子性**：所有事务更改在提交前被记录
- **持久性**：提交的更改在系统崩溃后仍然存在
- **恢复**：数据库可以恢复到一致状态

## WAL 结构

### WAL 文件布局

包含 WAL 头和 WAL 记录序列。

### WAL 记录格式

每条记录包含：
- 序列号
- 事务 ID
- 记录类型
- 时间戳
- 数据

## 写入过程

### 事务开始

记录事务开始，确保后续操作可以被回滚。

### 记录操作

将每个数据修改操作写入 WAL。

### 事务提交

写入提交记录并刷新到磁盘，确保持久性。

## 恢复过程

扫描 WAL 记录，分离已提交和未提交的事务，重放已提交的事务。

## 检查点

定期检查点以截断 WAL 并减少恢复时间。

## 性能优化

### 组提交

批量提交多个事务以提高吞吐量。

### WAL 缓冲

缓冲 WAL 写入以减少 I/O 操作。

## 最佳实践

1. **监控 WAL 大小**：在增长过大前检查点
2. **使用组提交**：批量提交以提高吞吐量
3. **调整缓冲区大小**：平衡内存和性能
4. **定期检查点**：防止长时间恢复
5. **监控恢复时间**：跟踪恢复需要多长时间
