# VitePress 自定义容器使用指南

本文档说明如何在 VitePress 文档中使用自定义容器和样式元素，以增强文档的可读性和视觉吸引力。

## 内置容器

VitePress 提供了多种内置的自定义容器，使用 `:::` 语法。

### Info 容器

用于提供补充信息、背景说明或详细解释。

```markdown
::: info 容器标题
这是信息内容。可以包含**粗体**、*斜体*、`代码`和其他 Markdown 元素。
:::
```

**渲染效果：**

::: info 容器标题
这是信息内容。可以包含**粗体**、*斜体*、`代码`和其他 Markdown 元素。
:::

**使用场景：**
- 提供背景信息
- 解释技术概念
- 补充说明细节

### Tip 容器

用于提供实用建议、最佳实践或优化技巧。

```markdown
::: tip 提示标题
这是一个有用的提示或建议。
:::
```

**渲染效果：**

::: tip 提示标题
这是一个有用的提示或建议。
:::

**使用场景：**
- 性能优化建议
- 最佳实践指导
- 快捷方式说明

### Warning 容器

用于提醒用户注意潜在问题或重要事项。

```markdown
::: warning 警告标题
这是一个警告信息，需要用户注意。
:::
```

**渲染效果：**

::: warning 警告标题
这是一个警告信息，需要用户注意。
:::

**使用场景：**
- 弃用警告
- 兼容性问题
- 配置注意事项

### Danger 容器

用于强调严重风险或关键限制。

```markdown
::: danger 危险标题
这是一个危险操作或严重限制。
:::
```

**渲染效果：**

::: danger 危险标题
这是一个危险操作或严重限制。
:::

**使用场景：**
- 数据丢失风险
- 安全漏洞警告
- 不可逆操作提示

### Details 容器

用于可折叠的详细内容，保持文档简洁的同时提供深度信息。

```markdown
::: details 折叠标题
点击展开查看详细内容。

这里可以包含任何 Markdown 内容。
:::
```

**使用场景：**
- 详细的技术规格
- 可选的高级配置
- 扩展阅读内容

## 徽章 (Badges)

虽然 VitePress 没有内置徽章组件，但可以使用 Markdown 和自定义样式创建。

### 复选框列表

用于列出已完成的功能或前置要求：

```markdown
::: tip 前置要求
在开始之前，请确保您已经：
- ✅ 编译了 Metrix 项目
- ✅ 生成了 `metrix-cli` 可执行文件
- ✅ 熟悉基本的 Cypher 查询语言
:::
```

**渲染效果：**

::: tip 前置要求
在开始之前，请确保您已经：
- ✅ 编译了 Metrix 项目
- ✅ 生成了 `metrix-cli` 可执行文件
- ✅ 熟悉基本的 Cypher 查询语言
:::

## 最佳实践

### 1. 选择合适的容器类型

| 容器类型 | 使用场景 | 示例 |
|---------|---------|------|
| `info` | 中性信息、背景说明 | 算法原理说明 |
| `tip` | 有用建议、优化技巧 | 性能优化建议 |
| `warning` | 注意事项、潜在问题 | 弃用警告 |
| `danger` | 严重风险、关键限制 | 数据丢失风险 |
| `details` | 可选详细信息 | 详细配置说明 |

### 2. 保持容器内容简洁

容器内容应该简明扼要，避免在容器中放置大量文本。

**好的做法：**

```markdown
::: info 什么是事务？
事务是一组作为单个逻辑单元执行的数据库操作。
:::
```

**不好的做法：**

```markdown
::: info
（此处放置 500 字的详细解释...）
:::
```

### 3. 使用有意义的标题

容器标题应该清楚地表明内容的性质。

**好的标题：**
- "前置要求"
- "性能优化建议"
- "已知限制"

**不好的标题：**
- "注意"
- "重要"
- "信息"

### 4. 适当嵌套容器

可以在容器中使用其他 Markdown 元素，包括列表、代码块、链接等。

```markdown
::: tip 配置建议
根据您的应用场景调整这些参数：

1. **高召回率优先**：增加 `beamWidth` 和 `maxDegree`
2. **速度优先**：减少 `beamWidth` 和 `maxDegree`
3. **内存受限**：减少 `maxDegree`，使用更激进的 PQ

参考 [参数调优](#parameter-tuning) 部分了解更多。
:::
```

### 5. 避免过度使用

不要在每个段落周围都使用容器。容器应该用于突出重要信息，而不是装饰所有内容。

**好的使用：**
- 关键概念说明
- 重要警告
- 实用建议

**不好的使用：**
- 普通描述性文本
- 常规代码示例
- 标准章节标题

## 代码示例中的容器

在代码说明中使用容器来强调关键点：

```markdown
### 插入向量

将新向量插入索引并构建图连接：

```cpp
void insert(int64_t nodeId, const std::vector<float>& vec) {
    // 1. 验证维度
    if (vec.size() != config_.dim) {
        throw std::invalid_argument("维度不匹配");
    }

    // 2. 达到阈值时自动训练 PQ
    if (!isPQTrained() && cachedCount_++ == config_.autoTrainThreshold) {
        auto samples = sampleVectors(config_.autoTrainThreshold);
        train(samples);
    }

    // ... 更多代码
}
```

::: tip 复杂度分析
**时间复杂度**：O(beamWidth × maxDegree × dim)
**空间复杂度**：O(maxDegree)
:::
```

## 常见模式

### 模式 1：前置要求检查清单

```markdown
::: tip 前置要求
在开始之前，请确保您已经：
- ✅ 安装了所有依赖
- ✅ 配置了环境变量
- ✅ 阅读了安全注意事项
:::
```

### 模式 2：配置参数说明

```markdown
::: details 配置说明
- **param1**：参数说明
- **param2**：参数说明
- **param3**：参数说明
:::
```

### 模式 3：最佳实践列表

```markdown
::: tip 性能优化建议
1. **建议 1**：详细说明
2. **建议 2**：详细说明
3. **建议 3**：详细说明
:::
```

### 模式 4：限制和注意事项

```markdown
::: warning 已知限制
- **限制 1**：说明
- **限制 2**：说明
- **限制 3**：说明
:::
```

### 模式 5：分步骤说明

```markdown
::: info 实现步骤
1. **步骤 1**：描述
2. **步骤 2**：描述
3. **步骤 3**：描述

每个步骤都应该...
:::
```
