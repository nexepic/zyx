# Rename "metrix" to "zyx" - Design Document

**Date:** 2026-03-17
**Status:** Approved
**Author:** Claude Code

## Overview

This document describes the design for renaming the Metrix graph database project to "zyx" (display name "ZYX"). This is a comprehensive renaming that affects the build system, public API, C API, documentation, and all user-facing strings.

## Approach

We will use **Approach 1: Build-Config Constants with Header Defines** to provide a single source of truth for project naming.

### Key Principles

1. **Centralized Configuration**: Define project names in one place (meson.build + template)
2. **Single Source of Truth**: Use `configure_file()` to generate headers at build time
3. **Consistent Naming**: Apply naming conventions across all components
4. **Breaking Change**: This is intentionally a major breaking change with no backward compatibility

## Architecture

### Configuration System

```
meson.build
    ↓
configuration_data() defines PROJECT_NAME="zyx", PROJECT_DISPLAY="ZYX"
    ↓
configure_file() reads ProjectConfig.hpp.in
    ↓
Generates buildDir/include/graph/core/ProjectConfig.hpp
    ↓
All code includes this header for project constants
```

### Data Flow

**Build-Time:** Meson generates configuration header from template
**Compile-Time:** Source files include generated header and use constants
**Runtime:** Applications display "ZYX" in CLI, prompts, and error messages

## Components

### New Files

1. **`include/graph/core/ProjectConfig.hpp.in`** (template)
   - Template file with `@PROJECT_NAME@`, `@PROJECT_DISPLAY@` placeholders
   - Defines both C macros and C++ constants
   - NOT compiled directly, used by `configure_file()`

2. **`buildDir/include/graph/core/ProjectConfig.hpp`** (generated)
   - Generated by Meson at build time
   - Contains actual constant values
   - Added to `.gitignore`
   - Included by code that needs project names

### Files to Modify

**Build System (3 files):**
- `meson.build` - Add `configure_file()` call
- `src/meson.build` - Update library names
- `apps/cli/meson.build` - Update executable name

**Public API (2 files + directory rename):**
- Rename: `include/metrix/` → `include/zyx/`
- `include/zyx/zyx.hpp` - Update namespace
- `include/zyx/zyx_c_api.h` - Update function/type names

**Internal Code (~5-8 files):**
- CLI files for display strings
- Error message files
- Any user-facing "Metrix" strings

**Documentation (~40-50 files):**
- All `.md` files in `docs/en/` and `docs/zh/`
- NexDoc documentation app configuration

## Naming Convention

| Context | Pattern | Example |
|---------|---------|---------|
| C++ namespace | lowercase | `namespace zyx` |
| C++ directory | lowercase | `include/zyx/` |
| C API functions | lowercase prefix + underscore | `zyx_open()` |
| C API types | ALL CAPS | `ZYXDB_T`, `ZYXValueType` |
| Display strings | ALL CAPS | "ZYX Database CLI" |
| Libraries | lowercase prefix | `libzyx`, `zyx_core` |
| CLI executable | single word | `zyx` |
| Benchmark executable | lowercase + suffix | `zyx-bench` |

### Detailed Mapping

**C API:**
- `metrix_open()` → `zyx_open()`
- `metrix_execute()` → `zyx_execute()`
- `MetrixDB_T` → `ZYXDB_T`
- `MetrixResult_T` → `ZYXResult_T`
- `MetrixValueType` → `ZYXValueType`
- `MetrixNode` → `ZYXNode`
- `MetrixEdge` → `ZYXEdge`

**Build Outputs:**
- `libmetrix.dylib` → `libzyx.dylib`
- `libmetrix_core.a` → `libzyx_core.a`
- `metrix-cli` → `zyx`
- `metrix-bench` → `zyx-bench`

**Display Strings:**
- "Metrix Database CLI" → "ZYX Database CLI"
- "metrix> " → "ZYX> "
- "<Metrix> Shell" → "<ZYX> Shell"

## Error Handling & Migration

### Breaking Changes

This is a **major breaking change**:

**Before:**
```cpp
#include <metrix/metrix.hpp>
metrix::Database db("/path");
```

**After:**
```cpp
#include <zyx/zyx.hpp>
zyx::Database db("/path");
```

### Build Failure Strategy

After renaming, a clean build is required:
```bash
rm -rf buildDir
./scripts/run_tests.sh --quick
```

### Rollback Strategy

- All changes in a single feature branch
- Git provides full rollback capability
- Can revert to previous commit if needed
